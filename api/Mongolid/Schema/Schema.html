Schema

class Schema

A schema maps to a MongoDB collection and defines the shape of the documents within that collection.

Properties

bool $dynamic The $dynamic property tells if the schema will accept additional fields that are not specified in the $fields property. This is useful if you does not have a strict document format or if you want to take full advantage of the "schemaless" nature of MongoDB.
string $collection Name of the collection where this kind of document is going to be saved or retrieved from.
string[] $fields Tells how a document should look like. If an scalar type is used, it will perform a cast into the value. Otherwise the schema will use the type as the name of the method to be called. See 'objectId' method for example.
string $entityClass Name of the class that will be used to represent a document of this Schema when retrieve from the database.
